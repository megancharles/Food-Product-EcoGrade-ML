Experiment 01 · March 2026

(Cleaner) ingredient data as a predictor of environmental performance

This experiment tests whether using structured ingredient data that has been processed using the Open Food Facts classification hierarchy, rather than relying on raw text or LLM-processed lists, produces cleaner inputs and improves the ability of a classifier to predict the eco-grade of a food product.

Dataset Open Food Facts · ~32,000 rows
Model XGBoost Classifier
Vectorisation TF-IDF
Data extracted February 2026
01

Background and hypothesis

During my PG Cert in Applied Data Science, I used an LLM to clean and standardise raw ingredient data. Whilst the processed outputs were cleaner than the raw text, they still contained two notable issues that are thought to have limited model performance.

The first was lack of standardisation across products. The same ingredient would appear in different forms depending on the product: "skimmed milk", "skimmed milk powder", "semi-skimmed milk", and "milk solids" might all refer to broadly the same ingredient category but be treated as distinct terms by a model. Similarly, "palm oil", "sustainable palm oil", and "RSPO-certified palm oil" would not be recognised as equivalent.

The second was inconsistent handling of nested ingredients. Many processed food products contain compound ingredients with their own sub-ingredient lists. For example, "chocolate chips (sugar, cocoa butter, cocoa mass, emulsifier: soya lecithin)" or "mayonnaise (rapeseed oil, egg yolk, vinegar, mustard)" each embed a second ingredient list within the first. LLM processing handled these inconsistently: sometimes flattening the sub-ingredients, sometimes preserving only the parent term, and sometimes duplicating information.

This experiment replaces LLM-processed data with ingredient data drawn directly from the Open Food Facts classification hierarchy, which provides a consistent, structured taxonomy for each ingredient. The hypothesis is that this more principled data source will reduce noise and improve classification accuracy.

Hypothesis

Cleaner, more consistently structured ingredient data will improve the ability of a classifier to predict eco-grade outcomes.

02

Methodology

01 Dataset

Approximately 32,000 rows were extracted from the Open Food Facts MongoDB data dump, combining two ingredient representations: raw ingredient text (ingredients_text), structured classification hierarchy (ingredients_hierarchy) containing all available parent and child values, and the green-score (ecoscore_grade).

Raw ingredient text "White Cabbage (45%), Rapeseed Oil, Carrots (15%), Water, Pasteurised Egg Yolk, Vinegar, Sugar, Onions, Potato Starch, Salt, Citrus Fibre, Dried Egg White, Dried Mustard."
→
Combined and classified "white cabbage, vegetable, brassica, cabbage, rapeseed oil, oil and fat, vegetable oil and fat, carrot, root vegetable, taproot vegetable, water, pasteurised egg yolk, egg, egg yolk, vinegar, sugar, added sugar, disaccharide, onion, onion family vegetable, potato starch, starch, salt, citrus fibre, fiber, vegetable fiber, egg white, mustard"
02 Class balancing

The dataset had imbalanced classes across the eco-grade spectrum. Training data was balanced by setting a minimum sample size equal to the smallest class (762 samples) and a maximum below the majority class (5,000 samples), giving each grade a fair representation during training.

03 TF-IDF vectorisation

Ingredient text was vectorised using TF-IDF (Term Frequency-Inverse Document Frequency). Because the data had already been processed through a classification hierarchy, each ingredient term carries its own meaning independently; the semantic context of surrounding words is not required. This made TF-IDF a more appropriate and computationally efficient choice than contextual embeddings such as BERT.

04 Modelling

An XGBoost Regressor was trialled first, but failed to predict the lowest eco-grade class (A+) reliably. The approach was switched to an XGBoost Classifier, which treats each grade as a distinct class and gives every category an equal opportunity to be predicted, making it better suited to this multi-class grading problem.
